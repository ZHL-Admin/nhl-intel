**PV-D017 | 2026-07-22 (Stage 3) | Per-fit defence-side sign convention (deny/suppress = ŌłÆdef_c, escape = +def_c).**
Each fit is the two-sided RAPM design with the ATTACKER as the `off` side and the DEFENDER as the `deff` side; the component is read off the centered DEFENCE coefficient `def_c`. deny and suppress are SUPPRESSION targets ŌĆö a good defender LOWERS the attacker's episode-start / in-zone-xG rate ŌĆö so the value is `-def_c` (identical to RAPM's `def_impact = -(centered def coef)`, higher = better). escape is a PRODUCTION target ŌĆö a favourable episode end (exit / flip_sustained) is the DEFENDER's success and RAISES the attacker-episode favourable-end rate ŌĆö so its value is `+def_c` (no flip). All three land "higher = better" (methodology ┬¦1). deny_rush is the diagnostic sub-fit, same sign as deny (`-def_c`); it is event-space only and never enters the assembled `pv_def_g60` (PV-D014). Rationale: the sign is dictated by whether good defence raises or lowers the modelled target, not by a blanket flip; getting escape's sign right is the one place a blanket `-def_c` would be wrong. **RATIFIED (owner, 2026-07-22 boundary review):** the escape no-flip is accepted as a correct intent-over-letter deviation from ┬¦7.2's blanket sign-flip language ŌĆö ┬¦7.2 assumes suppression targets; escape is a production target, so honouring ┬¦7.2's intent (higher = better) requires `+def_c`.
