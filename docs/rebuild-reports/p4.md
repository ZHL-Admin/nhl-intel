## 2. The two failures are environment artifacts, not defects

- **`compute_ratings`** (→ `team_ratings`): fails in `pd.merge_asof`
  (`add_trajectory_and_se`) on **every locally-available pandas** — pandas 3.0.3
  (research venv): incompatible datetime resolutions `[s]` vs `[us]`; pandas 1.5.3
  (a purpose-built py3.10 venv): a Cython `asof_join` ABI error. The code uses a
  version-specific `merge_asof(..., right_on=<Series>)` idiom written for the
  pandas the production Airflow image pins. **Not touched** (no behavior change).
  **Consequence:** `team_ratings` still holds **pre-retrain** values. The jobs that
  read it as a prior (`score_winprob`, `streak_doctor`, `simulate_deserved`,
  `roster_forecast`, `compute_team_needs`) ran green but on the **stale team prior**.
  → **Re-run `compute_ratings` + these five in the production env**, then re-export.
- **`compute_assessment`** (→ `player_assessment`): **computed successfully**
  (6,570 rows, estimator `c2_roster_player`) but the write failed on a **DuckDB
  serving-file lock** — a running backend process (`/opt/anaconda3` PID 11150) held
  `data/serving/nhl_intel.duckdb`. Pure lock contention. → **Re-run once the
  serving DB is free** (or stop the backend first).

Both are logged under `docs/rebuild-reports/p4_chain/`.

## 3. D9 staging fix (gated, applied) — recap

The sweep's first dbt pass failed 2 marts with `Bad int64 value`. Root cause:
1,129 degenerate 2019-20 shift rows (empty `endTime`, `duration 00:00`) whose
`shift_end_seconds` cast errored before the outer duration filter could drop them.
Fixed with a well-formedness guard in `stg_shifts` (`parsed` CTE), gated as
staging-only. **No-op assertion PASSED** — per-season row/duration/end checksums
byte-identical pre/post across all 16 seasons (15,943,699 rows), so the P1
reconciliation stands. Full detail: `docs/rebuild-reports/p4-blocker.md`; ledger
entry **D9** and the D8 concealment caveat are recorded in `upstream-ledger.md`.

## 4. Ratings-diff dossier + R1 flag

`docs/rebuild-reports/ratings-diff.md` contains, for the two most recent seasons
(2025-26, 2024-25): the 25 largest `player_composite` risers and fallers each
direction with a component-attributed one-line cause, league-level shift
statistics, and the retrained `player_impact` **YoY offense and defense Spearman**
(offense ρ 0.43–0.52, defense ρ 0.31–0.36).

Headline: the shift is a **systematic forward→defense redistribution** (2024-25
risers are almost all defensemen; fallers are elite forwards), matching the Atlas
R1 finding. Correlation to the pre-sweep ratings stays high (r 0.94–0.95).

**R1 flag (flag-only, no change made).** Three consumers project future value from
the adjusted lens with no raw component and so inherit the R1 −7.4% stayer
understatement: **`project_roster_forecast`**, **`compute_contract_value`**
(`blended_war_rate` core), and **`roster_player_projection`**. Recommendation (for
a future gated change): a conditional raw/adjusted blend keyed on the stayer/mover
distinction, blend weight fit against the R1 holdout. Descriptive consumers
(composite, gar, radar, deployment, verdict, impact-context) are **not** flagged.
